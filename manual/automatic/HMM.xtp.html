Configuration

HMM.xtp is configured through a UASR configuration file (parameter $2). The configuration file is a Java properties file containing key-value-pairs. The common configuration keys are described with the utility script cfg.itp (see there). Some commands use dedicated configuration keys which are described with the command. Dedicated keys may be placed in the configuration file just like all other keys. Any key (common and dedicated) may be specified or overwritten through the command line using -P<key>=<value> options:

 dlabpro HMM.xtp trn $UASR_HOME/config/vm/VMV.cfg -Pskip=SI -Pstart=2,5

-P options always overwrite the configuration file. The heading "uasr." of the keys may be omitted in -P options.

For further information

  • see documentation of util/cfg.itp to learn more about UASR's organization of databases.
  • see documentation of -CFG_get_path to learn how UASR determines paths to the various data sets.
  • see documentation of -CFG_get_flist to learn how UASR obtains file lists of the various data sets.
HMM.xtp adp Script Package Keyword Index Function Index Home
    Signature :    HMM.xtp adp <cfgfile> [options]
    Synopsis :    HMM adaptation
  • Description

Description

Adapts HMMs trained by HMM.xtp trn.

The MAP (Maximum a posteriori) adaptation method updates the HMM model parameters by combining prior known information (the old parameters) with the statistics derived from the adaptation data. In the first phase of the adaptation, the statistics required for computing the means and covariances are gathered from adaptation data. In the second phase, the gathered statistics are combined with the acoustic model statistics using a data-dependent weighting coefficient. The data-dependency of the weighting coefficients is realized by the relevance factor ρμ and ρΣ. Theirs values mark the point where the data count of the adaptation data has the same weight as the old parameter. Higher values of ρ gives more weight to the prior information i.e. the old parameters.

HMM.xtp trn Script Package Keyword Index Function Index Home
    Signature :    HMM.xtp trn <cfgfile> [options]
    Synopsis :    HMM training
  • Description
  • Program Flow Chart

Description

HMM.xtp trn is part of the UASR acoustic model training outlined in figure 1. It creates a set of HMMs from either

  1. a database consisting of labelled primary feature vector files or
  2. a database consisting of signal files and labels.

(See documentation of util/cfg.itp for details on the organization of databases in UASR.)

The first first method requires more disk space but is quicker if there is more than one HMM training iteration. The primary feature vector files database must be created before the actual model training

  • using FEA.xtp ana (primary feature analsis of signal files) or
  • using FEA.xtp imp (import of external feature files).

The second method saves the disk space for storing the primary feature vectors but it requires to compute these for each HMM training iteration anew.
