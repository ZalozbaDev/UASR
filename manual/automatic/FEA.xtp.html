Configuration

FEA.xtp is configured through a UASR configuration file (parameter $2). The configuration file is a Java properties file containing key-value-pairs. The common configuration keys are described with the utility script cfg.itp (see there). Some commands use dedicated configuration keys which are described with the command. Dedicated keys may be placed in the configuration file just like all other keys. Any key (common and dedicated) may be specified or overwritten through the command line using -P<key>=<value> options:

  
  FEA.xtp ana $UASR_HOME/config/vm/VMV.cfg -Pflist.all=myfilelist.txt

-P options always overwrite the configuration file. The heading "uasr." of the keys may be omitted in -P options.

For further information

  • see documentation of util/cfg.itp to learn more about UASR's organization of databases.
  • see documentation of -CFG_get_path to learn how UASR determines paths to the various data sets.
  • see documentation of -CFG_get_flist to learn how UASR obtains file lists of the various data sets.

Description

FEA.xtp is part of the UASR acoustic model training outlined in figure 1. It creates a set of HMMs or SVMs from either

  1. a database consisting of labelled primary feature vector files or
  2. a database consisting of signal files and labels.

(See documentation of util/cfg.itp for details on the organization of databases in UASR.)

The first first method requires more disk space but is quicker if there is more than one training procedure and/or iteration. The primary feature vector files database must be created before the actual model training

  • using FEA.xtp ana (primary feature analsis of signal files) or
  • using FEA.xtp imp (import of external feature files).

The second method saves the disk space for storing the primary feature vectors but it requires to compute these for each training procedure/iteration anew.

FEA.xtp cpy Script Package Keyword Index Function Index Home
    Signature :    FEA.xtp cpy <cfgfile> [options]
    Synopsis :    Copying of feature files
  • Description

Description

Copies feature files created by FEA.xtp ana or FEA.xtp imp to the directory specified by the configuration key uasr.dir.out. The program copies the files belonging to the data set specified by the cofiguration key uasr.dataset. If this key is not specified, the files of the data set named "all" will be copied. See the documentaion of -GFG_get_flist for a list of valid data set identifiers.

Start and Command Line Options

Configuration

Important common keys

Dedicated and important common configuration keys

Examples

  
  FEA.xtp cpy myconfig.cfg -Pdir.out=mydir                 # Copy all files to mydir
  FEA.xtp cpy myconfig.cfg -Pdir.out=mydir -Pdataset=train # Copy the training set only 
  FEA.xtp cpy myconfig.cfg -Pdir.out=mydir -n -v2          # Say what to do but do not copy any files

FEA.xtp imp Script Package Keyword Index Function Index Home
    Signature :    FEA.xtp imp <cfgfile> [options]
    Synopsis :    Import of external feature files
  • Description

Description

Imports and labels primary feature vector files stored in the database in a third-party format. For this program to do something useful you must overwrite the UASR functions -FEA_import and, occasionally, -LAB_import to provide appropriate import filters. Place the overwrites in the database dependent include file or use the configuration key <uasr.customize> in order to load a customization script which contains the overwrites.

For details on the UASR model training scheme see section Description above.
