Function -HMM_save Functions Script Package Keyword Index Function Index Home
    Signature :    -HMM_save(sFile,itHMM)
    Source :    train/am.itp(51)

Description

Saves an HMM instance to a zipped DN3 file. The function removes the feature statistics' before saving the HMM to spare disk space. An HMM instance saved by this function may be restored using dLabPro's -restore method for recognition. For further training, however, you must use the -HMM_load function to restore the HMM instance.

Parameters

   string    sFile   Path name of HMM file
   hmm    itHMM   HMM instance to save

Return value

TRUE if successful, FALSE otherwise

Function -HMM_assess Functions Script Package Keyword Index Function Index Home
    Signature :    -HMM_assess(itHMM,sSns,nSpl,nIte)
    Source :    train/am.itp(529)

Description

Decodes all files in the test set through all HMMs. This yields a vector of neglog likelihoods (NLLs), one for each HMM, per test file. The NLL values are normalized to the length of the test file (i.e. to the number of feature vectors). All these vectors will be concatenated to a matrix whose rows (components) contain the NLLs for one HMM and whose columns (records) contain the NLLs for one test file. The function saves this matrix as a file named hmm_assess-<HMM-id>.dn3 in the log directory (configured through <uasr.dir.log>). It also saves one NLL histogram per HMM in files named hmm_assess-<HMM-id>-<unit-id>.dn3. If the evaluation file list (configured through <uasr.flist.test>) contains a label component, the function will classify each test file by assigning it the label of the HMM giving the lowest NLL for it. By comparing the detected labels to the labels known from the file list, the function computes a label recognition rate which is returned. If the evaluation file list does not contain labels the function returns 1.

This function may overwrite -HMM_eval:

 
    /disarm -HMM_assess /disarm -HMM_eval =;

   number    nSpl   Number of state splits of HMM to be assessed
   number    nIte   Update iteration of HMM to be assessed

Return value

The label recognition rate or 1 if no correct labels are known.

Function -HMM_process Functions Script Package Keyword Index Function Index Home
    Signature :    -HMM_process(itHMM,sSns,nSpl,nIte,sDatSet,sResFnc,sMode)
    Source :    train/am.itp(1187)

Description

Decodes a data set of signals (turns) through a recognition network using the Viterbi- or forward-backward-procedure and passes the decoding results to the user defined function sResFnc. The recognition network is built by composing the Kleene closure of all HMMs contained in itHMM with the recognition grammar obtained through -GRM_get.

Result Function sResFnc

The result function is called from -HMM_process before, during and after processing the turn list. Its signature is

  
  function -HMM_process_result(itHMM,sSns,nSpl,nIte,sHint,iF,itRes,iL)

and its parameters are

Type IdentifierDescription
hmm itHMMThe HMM set (as described below)
stringsSns Sensor ID (as described below)
numbersSpl Split number (as described below)
numbersIte Iteration number (as described below)
stringsHint One of the following:
"I"The first call. The function is expected to return a human-readable text identifying its purpose (e.g. "Evaluation" or "Training iteration").
"F"The second call, before the decoding of the first turn of the data set. On this call the function may prepare the container object iL for later use during the processing of individual decoding results, e.g. by creating and initializing data, var or other instances in the container.
""Called once per turn with itRes containing the decoding result for that turn. The function may use the data inside the container iL initialized in the first place.
"L"The last call, after the decoding of the last turn of the data set. The function is expected to return a real number (preferrably between 0 an 1) indicating the success of the HMM process. On evaluations, this return value might be the product of correctness and accuracy.
fileiFA file list. Its current file specifies the turn being decoded.
fstitResThe decoding result (only if sHint="")
objectiLContainer object for the use of the evaluation function. This instance will be committed on every call to the result function except the very first one (sHint="I").

Known result functions are:

  • -HMM_eval_fnc
  • -HMM_label_fnc

Examples

1. Using -HMM_process in a script

The following program performs a Viterbi-evaluation of the HMM "0_10" (split 0, iteration 10; a configuration file is to be passed as first argument to the script):

   
   "$UASR_HOME/scripts/dlabpro/util/@.itp"   include;
   "$UASR_HOME/scripts/dlabpro/train/am.itp" include;
   "$1" TRUE -CFG_init;
   "$UASR_HOME/scripts/dlabpro/util/uasr_session.itp" include;
   
   hmm itHMM;
   itHMM NULL 1 10 "test" "-HMM_eval_fnc" "TV" -HMM_process;
2. Modifying the evaluation procedure of HMM.xtp trn

HMM training uses -HMM_process for the model evaluation. To implement a custom procedure you need to overwrite the result function, e.g. in your database include script:

   
   function -MY_hmm_eval_func(itHMM,sSns,nSpl,nIte,sHint,iF,itRes,iL)
   {
     sHint "I" == if "My procedure" return; end
     sHint "F" == if
       # Do initialization stuff here
       0 return;
     end
     sHint "L" == if
       # Do clean up stuff here
       1 return;
     end
   
     # Per turn evaluation stuff goes here
     0 return;
   }
   
   /disarm -MY_hmm_eval_fnc /disarm -HMM_eval_fnc =;

   number    nSpl   Number of state splits of HMM, may be -1

Function -XXM_assess_log Functions Script Package Keyword Index Function Index Home
    Signature :    -XXM_assess_log(sFilePref,sFileSuff,idLtb,idFlst,idAsc)
    Source :    train/am.itp(1516)

Description

Creates or continues the assessment log file and the label confusion matrix. Both files will be written into the log folder. The log file name is "<sFilePref>_assess<sFileSuff>.dn3"; the confusion matrix file name is "<sFilePref>_assess-cmx<sFileSuff>.dn3". The log file will always be written, the confusion matrix will only be writte if the configuration key <uasr.am.eval.assess> contains the string "cmx".

Present log files will be continued if and only if the configuration key uasr.xvl is TRUE. Otherwise present log files will be overwritten. Please be advised that the uasr.xvl key is not intended to be used in configuration files or on the command line.

Parameters

   string    sFilePref   Prefix identifying the model ("hmm-<spl>-<ite>" for HMMs, "svm" for SVM, etc.)
   string    sFileSuff   Suffix identifying the sensor
   data    idLtb   Label table (component 0 must contain the symbolic labels)
   data    idFlst   The assessment file list with the following component structure:
Comp.Type Description
0 stringFile name
1 stringFile label (optional)
   data    idAsc   The assessment result object (created within -HMM_assess and -SVM_training.xtp) The instance is expected to contain at least idLtb.nrec numeric components and exactly idF.nrec records. Extra components will be ignored. The cells idAsc[nF,nM] contains a distance value (e.g. neglog. likelihood or neglog. probability) of file nF from the model nM.

Return value

The label recognition rate or -1 if no reference labels are known.
