Synopsis

Training functions: HMM topology training

Description

The functions in this file can be used to train the topology of models. To activate them use the configuration keys uasr.am.train.act[0-9], uasr.am.train.acta[0-9]_[0-9], uasr.am.train.actb[0-9]_[0-9] (see Configuration keys). Following actions are implemented:

NameFunctionDescription
split-HMM_Act_split Split a defined number of Gaussians
hmm-HMM_Act_hmm Convert models using -hmm function
mix-HMM_Act_mix Convert hmm to mixture Gaussians
tprun-HMM_Act_TPrun Simple transition pruning using weight treshold
prun-HMM_Act_Prun Pruning of transitions according the path weights of thier surroundings
uprc-HMM_Act_UsedPathRC Pruning of unused transition (defined by dev data set)
bestn-HMM_Act_BestN Reduce model to best paths (neg.-log. density as selection criterion)
up-HMM_Act_UsedPath Reduce model to used paths (defined by dev data set)

These function use the following configuration keys:

The functions can be used in different scenarios. One scenario is transition purning after the third split and four more iterations of parameter estimation. For this example here is a configuration file, which uses used path transition pruning with 8000 transitions to remain in the model after pruning. 

	## HMM training settings
	uasr.am.train.split       = 5;
	uasr.am.train.ite0        = 5;
	uasr.am.train.ite1        = 10;
	uasr.am.train.ite2        = 15;
	uasr.am.train.ite3        = 20;
	uasr.am.train.ite4        = 4;
	uasr.am.train.act3        = "uprc";
	uasr.am.train.uprc.num    = 8000;

A second scenario is topology training of technical signals. For izfp valve signals the following configuration file may be used. It trains the topology by alterately spliting of pruning the model. The spliting is done by selection the half of all Gaussians according thier average density. The pruning is done by pruning of unused paths. Between all these topology optimizing one iteration of parameter adaption is executed. In the last step by usedpath function the horziontal topology of the signals is determined. 

	## HMM training settings
	## Configure training schedule
	uasr.am.train.split     = 9;
	uasr.am.train.ite0      = 0;
	uasr.am.train.ite1      = 1;

	## Use simple transition pruning after every update
	uasr.am.train.acta0_0   = "tprun";
	uasr.am.train.acta1_0   = "tprun";
	uasr.am.train.acta2_0   = "tprun";

	uasr.am.train.acta4_0   = "tprun";

	uasr.am.train.acta6_0   = "tprun";
	uasr.am.train.acta7_0   = "tprun";
	uasr.am.train.acta7_1   = "tprun";
	uasr.am.train.acta8_0   = "tprun";
	uasr.am.train.acta8_1   = "tprun";
	uasr.am.train.acta9_0   = "tprun";
	uasr.am.train.acta9_1   = "tprun";
	uasr.am.train.acta9_2   = "tprun";

	## Alternately split or prun the model
	uasr.am.train.act0      = "split,hmm";
	uasr.am.train.act1      = "uprc";
	uasr.am.train.act2      = "split,hmm";
	uasr.am.train.act3      = "uprc";
	uasr.am.train.act4      = "split,hmm";
	uasr.am.train.act5      = "uprc";
	uasr.am.train.act6      = "split,hmm";
	uasr.am.train.act7      = "uprc";
	uasr.am.train.act8      = "up";

	## Config used methods
	uasr.am.train.tprun.thresh = 1000;
	uasr.am.train.split.krit   = "pd";
	uasr.am.train.split.num    = -0.5;
	uasr.am.train.uprc.num     = -2;
	uasr.am.train.up.num       = 50;

There are three different selection criterions ("var"-Variance, "rc"-Reference Counter, "pd"-Probability density, "ipd"-Inverse probability density).
