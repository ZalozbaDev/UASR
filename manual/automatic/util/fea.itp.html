Synopsis

Utility functions: Primary and secondary feature analysis and synthesis

Description

Computing feature vectors from a signal is a two stage process comprising a primary and a secondary analysis (Fig. 1 shows an outline). Reconstructing the signal from the features is, in principle, the inverse operation. However, as the analysis may reduce the feature dimensionality, additional information may be required for signal (re)synthesis.

 

Figure 1: Overview of UASR's feature analysis

Primary Feature Analysis and Synthesis

UASR uses short-term signal transforms and their inverses for primary feature analysis and synthesis. See documentation of configuration key <uasr.pfa> for a list of available short-term transforms. Analysis and synthesis are implemented by the functions -FEA_pfa and -FEA_ipfa respectively.

Secondary Feature Analysis and Synthesis

The secondary feature analysis performs some or all of the following functions:

  • vector standardization
  • computation of delta, delta-delta, and context features
  • FIR filtering
  • linear transforms and dimension reduction

See documentation of configuration key <uasr.sfa> for a list of available options. Analysis and synthesis are implemented by the functions -FEA_sfa and -FEA_isfa respectively.

Function -FEA_info Functions Script Package Keyword Index Function Index Home
    Signature :    -FEA_info(sFile,iFI)
    Source :    util/fea.itp(96)

Description

Loads a feature information object or creates an initial one. The feature information object controls the secondary feature transformation (see function -FEA_sfa).

The feature information object is structured as follows:

iFI Symbol  Description
  var nDim  N Feature space dimension, sum of most and least significant dimensions
  data idDlt  Δ Delta computation table (dimension N×2, neutral: zero-matrix)
  data idX  x0 Feature offset vector (dimension N, neutral: zero-vector)
  data idW  W Linear transformation matrix (dimension N×N, neutral: identity matrix)
  data idPrz   Feature vector component priorization info, e.g. Eigenvalues in case of PCA or neg. entropies in case of ICA

If sFile is NULL or an empty string, the function creates a neutral feature information object, i.e. passig it to -FEA_sfa will leave the input feature vectors unchanged.

Remarks

  • The standard secondary feature transformation is y = W · (delta(x,Δ) + x0), where x denotes the original and y the transformed feature vector.

Parameters

     sFile   File name of feature information object to be restored. If NULL or empty, a neutral feature information object will be created
   object    iFI   An empty object to be initialized
Function -FEA_delta_tab Functions Script Package Keyword Index Function Index Home
    Signature :    -FEA_delta_tab
    Source :    util/fea.itp(164)

Description

Creates a delta computation table. Non-zero values in the first(second) record of the table define feature vector components for which delta(delta-delta) features shall be computed. For details see documentation of dLabPro method FBAproc -delta.

The function is entirely controlled by the session configuration. In any case it returns a table with <uasr.fea.pfa.dim> components (i.e. the dimension of the primary feature vector) and 2 records. The following types of delta tables are supported.

<uasr.sfa.delta>="none"

No delta/delta-delta features will be computed. In this case the function returns a blank table.

<uasr.sfa.delta>="pattern"

The delta table will be built repeating patterns for deltas and delta-deltas. The following configuration values are used (all may be omitted):

KeyTypeDefaultDescription
uasr.sfa.delta.dpatstring"1 0" Delta pattern, white space sepatated list of ones and zeros
uasr.sda.delta.dofsnumber0 Offset of delta pattern (component index to start pattern at)
uasr.sfa.delta.drepnumber0 Number of repetitions of delta pattern (0 for infinite repetition)
uasr.sfa.delta.ddpatstring"1 0" Delta-delta pattern, white space sepatated list of ones and zeros
uasr.sfa.delta.ddofsnumber0 Offset of delta-delta pattern (component index to start pattern at)
uasr.sfa.delta.ddrepnumber0 Number of repetitions of delta-delta pattern (0 for infinite repetition)

Return value

The delta computation table

Function -FEA_get Functions Script Package Keyword Index Function Index Home
    Signature :    -FEA_get(iF,sSns,sMode,idFea)
    Source :    util/fea.itp(296)

Description

Obtains the primary feature vector sequence for one turn and sensor. The turn is defined by the current position of the file list iF. The function is provided for conveniently iterating data sets.

Example

   
  file iF;
  data idFea;
  ( "test" "" -CFG_get_flist ) iF -set flist;
    label L_F; iF -next if
    iF NULL "" idFea -FEA_get;
    ...
    goto L_F;
  end

Remarks

In the (default) automatic mode (sMode="A"), the function proceeds as follows:

  • Try to load the feature vector file stored in <uasr.dir.fea>.
  • If no such file exists:
    • Call -SIG_get,
    • Only if <uasr.fea.pm> is TRUE: load the pitch mark file stored in <uasr.dir.sig> or, if no such file exists, computes pitchmarks,
    • Perform the primary feature analysis by calling -FEA_pfa, and
    • Only if <uasr.fea.f0> is TRUE: append the mean fundamental frequency as additional feature vector component.
  • If neither a feature vector nor a signal file exists: clear idFea and return FALSE, otherwise return TRUE.

Parameters

   file    iF   File list instance
   string    sSns   Sensor ID, NULL or empty for one-channel recordings (typical case for speech recognition)
   string    sMode   Exactly one of the following characters
Character Description
ADefault behavior (see above)
FLoad feature file
SLoad signal file and perform primary feature analysis
If empty or invalid, "A" is assumed.
   data    idFea   Will be filled with the feature vector sequence

Return value

TRUE if successful, FALSE otherwise.

Function -FEA_import_raw Functions Script Package Keyword Index Function Index Home
    Signature :    -FEA_import_raw(sFile,idFea)
    Source :    util/fea.itp(512)

Description

Imports and compresses a raw spectrogram file. The raw file is expected to contain a sequence of <import.raw.dim>-dimensional vectors. The file starts with the values of the first vector, continues with the values of the second vector and do forth. This function may overwrite -FEA_import: 
 
  /disarm -FEA_import_raw /disarm -FEA_import =;

Remarks

  • The following data must be provided through configuration keys:
    Information Configuration Key Default
    Byte order import.raw.byteorder "" (meaning "do not change")
    Increment between two channels (spectral lines), default 1 import.raw.cinc 1
    Physical unit of channel (spectral, component) axis; default: "kHz" import.raw.cunit "kHz"
    Number of channels in raw file import.raw.dim <uasr.pfa.dim>
    Number of records to import import.raw.nrec -1 (meaning "all")
    Raw data values' type import.raw.dtype n/a
    Raw header size [bytes] import.raw.header 0
    Time increment between two feature vectors [ms], default 10 import.raw.rinc 10
    Output feature vector dimension pfa.dim n/a
  • On verbose level 3 and higher (global variable .__UTL_nVlevel) the function displays a message including length of the imported feature vector sequence in milliseconds.
  • The function reduces the spectral dimension from <import.raw.dim> to <pfa.dim> by averaging adjacent spectral lines.
  • <import.raw.dim> must be an integer multiple of <pfa.dim>.

Parameters

Function -FEA_pfa_lmag Functions Script Package Keyword Index Function Index Home
    Signature :    -FEA_pfa_lmag(idSig,idPM,idFea)
    Source :    util/fea.itp(825)

Description

Primary feature analysis through short-term log-magnitude spectrum. Figure 1 shows what the function does in detail.

Figure 1: Computation of short term (log-) magnitude spectrum

This function may overwrite -FEA_pfa:

 
 /disarm -FEA_pfa_lmag /disarm -FEA_pfa =;

Remarks

  • When used for the first time in a session, the function calls -FEA_pfa_init("FFT") in order to initialize the underlying FFTproc. Most of the configuration values and global variables listed below are accessed from -FEA_pfa_init rather than from -FEA_pfa_lmag.
  • The output dimension is clipped to 2<uasr.pfa.FFT.order>-1 or <uasr.pfa.dim>, whichever is smaller.
  • The trailing (<uasr.pfa.wlen> / <uasr.pfa.crate>) -1 vectors will be trimmed (as their zero padding distorts the spectrum).

Function -FEA_priorize Functions Script Package Keyword Index Function Index Home
    Signature :    -FEA_priorize(iF,sSns,iFI,idXca,idNent)
    Source :    util/fea.itp(1128)

Description

Permutates a linear feature transformation (PCA or ICA) matrix such that applying the transformation will yield components ordered by descending negative entropy. The function estimates the skewness and excess of the feature vectors transformed through idXca from the specified file list. Before transforming the features through idXca, all transformations specified by iFI (i.e. delta computation, normalization, PCA and ICA) will be applied.

Remarks

  • If you use the function to priorize the PCA or ICA transformation matrices in iFI make sure to reset the respective instance in iFI prior to calling the function and pass its contents separately as idXca. Otherwise the feature vectors will be multiplied with the transformation matrix twice!

Parameters

   file    iF   File list defining a set of feature files to gather statistics from
   string    sSns   Sensor ID, NULL or empty for one-channel recordings (typical case for speech recognition)
   object    iFI   Feature information object
   data    idXca   Linear feature transformation matrix, will be permutated
   data    idNent   Will be filled with the negative entropies of the vector components after permutation, may be NULL
