Function -FST_lvnstn Functions Script Package Keyword Index Function Index Home
    Signature :    -FST_lvnstn(itSrc1,itSrc2,nUnit1,nUnit2,idCost,idConf,idDst)
    Source :    util/fst.itp(81)

Description

Computes the Levenshtein distance between two finite state machines.

Parameters

   fst    itSrc1   Automaton 1
   fst    itSrc2   Automaton 2
   number    nUnit1   Unit index of automaton 1
   number    nUnit2   Unit index of automaton 2
   data    idCost   Symbol match cost matrix. May be NULL in order to use the standard costs (3 for insertions/deletions and 4 for subsitutions). Otherwise idCost is expected to be a N-by-N matrix (where N is the size of the transducer input alphabet excluding the epsilon symbol) specifying the symbol match costs. idCost.descr0 must contain the insertion and deletion cost.
   data    idConf   Symbol confusion matrix (may be NULL). The matrix is expected to contain at least X1+1 records and X2+1 components (all numeric), where X1 denotes the size of the input alphabet of itSrc1 and X2 the size of the input alphabet of itSrc2, respectively. The values in record i count the input symbols from itSrc2 which where matched on the input symbol xi from itSrc2. The values in component j count the input symbols from itSrc1 which where matched on the input symbol xj from itSrc1. The last record/component counts symbols matched on the epsilon symbol, i.e. which where deleted.
   data    idDst   Data instance to be filled with detailed result (may be NULL).

Return value

number The Levenshtein distance or a negative number in case of errors.
Function -FST_render Functions Script Package Keyword Index Function Index Home
    Signature :    -FST_render(sFname,idTisTab,idTosTab,nUnit,itSrc)
    Source :    util/fst.itp(240)

Description

Renders an FST as SVG graphic.

Remarks

  • The function requires the dot utility by AT&T. The path environment variable should include the location of this program.
  • The function will not render other transition qualifications than input symbols, output symbols and weights!
  • The function will order the transition qualifications as follows: 1. input symbols, 2. output symbols, 3. Weights. If one or serveral of these are not present, the function will omit them.

Parameters

   string    sFname   Name of output file (relative or full path)
   data    idTisTab   Lookup table of transducer input symols, may be NULL. Function will replace the input symbol indices by symbols looked up from the first component of idTisTab for rendering.
   data    idTosTab   Lookup table of transducer output symols, may be NULL. Function will replace the output symbol indices by symbols looked up from the first component of idTosTab for rendering.
   number    nUnit   Zero-based index of FST-unit to be rendered.
   fst    itSrc   FST to be rendered.

Return value

void

Description

Renders an FST as graphic.

Remarks

  • The function requires the dot utility by AT&T. The path environment variable should include the location of this program.
  • The function will not render other transition qualifications than input symbols, output symbols and weights!
  • The function will order the transition qualifications as follows: 1. input symbols, 2. output symbols, 3. Weights. If one or serveral of these are not present, the function will omit them.

Parameters

   string    sFname   Name of output file (relative or full path)
   data    idTisTab   Lookup table of transducer input symols, may be NULL. Function will replace the input symbol indices by symbols looked up from the first component of idTisTab for rendering. If it is NULL the input symbol table of itSrc is used.
   data    idTosTab   Lookup table of transducer output symols, may be NULL. Function will replace the output symbol indices by symbols looked up from the first component of idTosTab for rendering. If it is NULL the output symbol table of itSrc is used.
   number    nUnit   Zero-based index of FST-unit to be rendered.
   string    sFormat   One of the following
sFormatDescription
"png" Portable Network Graphics (pixel)
"ps" Postscript Graphics (EPSF, vector)
"svg" Scalable Vector Graphics (vector)
   fst    itSrc   FST to be rendered.

void
