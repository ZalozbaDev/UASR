Function -AM_decode Functions Script Package Keyword Index Function Index Home
    Signature :    -AM_decode(idSrc,iFI,itAM,nUnit,sMode,sTID,itDst)
    Source :    util/am.itp(186)

Description

Decoder. There are four possbile types of input data and two possible types of output data (both to be specified through sMode).

Parameters

   data    idSrc   Source data as specified by nMode (see table above)
   object    iFI   A feature information object (may be NULL in modes 2 and 3)
   hmm    itAM   The Hidden Markov model to be used for decoding. The automaton graph is expected to describe an appropriate decoding network. If itAM contains more than one unit, the function will decode the input through all networks and store the results in separate units of idDst.
     nUnit   Unit to process (may be -1 for all units).
   string    sMode   A string of two characters, the first specifies the input data type committed through idSrc, the second specifies the output data type to be stored in itDst. A third character may be given and specifies the search algorithm.
1st char.Description
0Signal (one numeric component)
1Primary feature vector sequence
2Secondary feature vector sequence
3Gaussian neg. log. density array
2nd char.Description
AUnprocessed path in deconding network
OChain of output symbols
LOnly load search processor for futher invocations with the same automaton
FOnly free loaded search processor
3nd char. (optional)Description
DUse dynamic programming (default)
TUse token passing search
AUse synchron a* search
FUse forward-backward algorithm
   string    sTID   Unique turn ID of source data for logs and dumps, can be NULL
   fst    itDst   The recognition result
