Function -LX_load Functions Script Package Keyword Index Function Index Home
    Signature :    -LX_load(sFile,sFilter,idLex)
    Source :    util/lx.itp(79)

Description

Loads a pronunciation lexicon from a text file. Each line in the file defines one lexicon entry. Pronunciation variants may be defined giving multiple entries with identical orthographic strings. One entry consists of at least two white space separated values: the orthographic and phonetic strings. An optional third column may contain a word weight. In case of pronunciation variants only the weight given for the first variant will be used. Two further optional columns may contain increments of the rejection thresholds.

Example (sFiler="LEX:")

  
   FOO: what ever ...
  
   LEX: eins  QaIns
   LEX: eins  aIns
   LEX: zwei  tsvaI 10
   LEX: zwei  tsvo:
  
   BAR: what ever ...

See also

  • -LX_compile

Parameters

   string    sFile   Path of text file to be loaded
   string    sFilter   Line filter (consider only lines starting with the given string), commit "" to read all lines
   data    idLex   Will be filled with the lexicon entries

Return value

TRUE if successfull, FALSE otherwise

    Source :    util/lx.itp(306)

Description

Compiles a lexicon from a string list into a transducer.

Parameters

   data    idPtab   Phoneme table (list of phoneme symbols expected in component 0)
   data    idLX   The lexicon (list of words) to be compiled. The list is expected to contain the following components:
ComponentDescription
0Mandatory: orthographic strings
1Mandatory: phonetic strings
2Optional: Word weigth (log. semiring), 0: neutral
3Optional: NAD confidence threshold, 0: use default
4Optional: NED confidence threshold, 0: use default
   fst    itLX   Instance to be filled with the lexical transducer (one FST unit per record in idLX)

Return value

nothing
Function -LX_get Functions Script Package Keyword Index Function Index Home
    Signature :    -LX_get(idAM,itLX)
    Source :    util/lx.itp(357)

Description

Loads the configured lexicon. The function proceeds as follows:
  1. Load the lexicon file specified through configuration key uasr.lx (if present).
  2. If the language model is a finite state grammar (configuration key uasr.lm="fsg"), additionally load lexicon entries from grammar file specified through configuration key uasr.lm.fsg.
  3. Add the pause word "<PAU>" and the filler word "<OOV>".

See also

  • -LX_load

Parameters

   data    idAM   Acoustic models (list of terminal lexicon symbols, e.g. list of phonemes). The list of acoustic models can be obtained through -LAB_get_tab. The unit table hmm.ud of a Hidden Markov model set may be committed, too.
   fst    itLX   Filled with the lexicon

Access to Configuration and Global Instances

   <uasr.am.classes>    read 
   <uasr.am.gbg>    read 
   <uasr.am.sil>    read 
   <uasr.info>    read 
   <uasr.lm>    read 
   <uasr.lm.fsg>    read 
   <uasr.lx>    read 
Function -LX_unify Functions Script Package Keyword Index Function Index Home
    Signature :    -LX_unify(itSrc,itDst)
    Source :    util/lx.itp(398)

Description

Unifies and minimizes a lexical transducer.

Remarks

Some lexicon entries (like the filler model "<OOV>") are not minimizable. Hence idSrc [...] -minimize will fail. This function only minimizes ordinary entries (e. g. words whose names do not begin with "<") and leaves all other entries alone.

Parameters

   fst    itSrc   Source transducer, is expected to contain one unit per lexicon entry
   fst    itDst   Filled with a phoneme-to-word transducer, may be indentical with itSrc
