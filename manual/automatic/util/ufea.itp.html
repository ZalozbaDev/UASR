Synopsis

Universal Primary and secondary feature analysis and synthesis

Universal primary feature analysis

The idea of the universal primary feature analysis is to combine multiple analysis methods into one unified dataflow. All methods share the same, common pre- and postprocessing. To use universal analysis you have to set the configuration key uasr.pfa="UPFA". The figure 1 shows the implemented dataflow as well as all important configuration keys. The configuration keys are printed as subkeys of uasr.pfa.UPFA.

A detailed description of the keys and their default values can be found under UASR Configuration Files. There in the default column you find classic PFA configuration keys, these are fallbacks. If the UPFA specific configuration key is not defined, the fall back key of classic PFA is read. Only if that key is also not defined, the default value is used.

 

Figure 1: Dataflow for universal primary feature analysis

The UPFA consists of the following analysis components:

  • The Mixing can be used the mix different signal channels together. The gain and the delay can be configured, so a simple static beam former is also possible. In default mode only the first channel is processed without modification.
  • The RMDC component removes the mean of the signal. It is disabled by default
  • The Framing component is used the split the signal into frames. It is done pitch synchronous if pitches are provided to -UPFA_ana, otherwise a static window length (uasr.pfa.UPFA.wlen) and continuous rate (uasr.pfa.UPFA.crate) is used.
  • The Window component multiplies each frame sample by sample with a window function. In addition it expands the frames to full frame length (uasr.pfa.UPFA.len). Finally the frames are added with quantization noise.
  • The Transformation component preforms the core analysis method. Currently FFT, Wavlet and generalized MEL-Cepstrum are implemented. You can define your own methods by implementing function of name -UPFA_trf_<NAME>.
  • The first Mapping component is used for postprocessing the analysis result. It preforms a customizable mapping on each value in each frame. It is used for example to get amplitude spectrum from complex FFT by calculation absolute values. You can define your own mapping methods by implementing function of name -UPFA_map_<NAME>.
  • The Filtercomponent multiplies each frame with a configurable filter matrix. It is used for example for MEL-filter. But may also be used for a custom spatial mapping or simply for selecting features. You can define your own filter matrix by implementing function of name -UPFA_init_fil_<NAME>.
  • The Dimension component checks if the configured frame dimension uasr.pfa.UPFA.dim or uasr.pfa.dim is met. It clips or expands the frame dimension if necessary.
  • The second Mapping component is used for postprocessing the filter result. It preforms a customizable mapping on each value in each frame. It is used for example to get logarithmic features. You can define your own mapping methods by implementing function of name -UPFA_map_<NAME>.

The figures 2 and 3 give a more detailed view on the transformation and the filter component of UPFA. There are also some configuration keys which are specific for certaint algorithms.

Migrating to UPFA

For migration from dlabpro's feature processors to UPFA you need to set the transformation, the mappings and the filter method in UPFA. The following table gives an overview over settings which are compatible between old and universal analysis. The generated features are nonetheless not equal because the way of adding quantization noise was changed for universal analysis.

Function -UPFA_map_ABSLOG Functions Script Package Keyword Index Function Index Home
    Signature :    -UPFA_map_ABSLOG(idFea)
    Source :    util/ufea.itp(96)

Description

UPFA mapping function taking the logarithm.

This function replaces each value in idFea with the natural logarithm of the absoult value. Finally the value is bottom clipped to quantization error logarithm: -ln(idFea.nScale).

The function is used for preprocess by MEL-filter with Sinc transfer function.

Parameters

   data    idFea   Current feature data object

Description

UPFA mapping function taking the logarithm.

This function replaces each value in idFea with the natural logarithm of this value. Finally the quantization error logarithm [-ln(idFea.nScale)] is added and the result is bottom clipped to zero.

The function can be used after filter to get logarithmic features.
