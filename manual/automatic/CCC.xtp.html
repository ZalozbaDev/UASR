Usage

  dlabpro CCC.xtp help [<command>]
 
Argument/OptionDescription
Arguments
$1: command Command, one of the following:
 
cls - Classification
trn - Training
ren - Auxiliary: Render spectral windows

or "help"
$2: cfgfile UASR configuration file
 
Options
OptionDescription
-h, --help Display help and exit
-P<key>=<value> Define or override configuration key-value pairs
-v<n> Verbose level
-v1   minimal screen protocol (default)
-v2extended screen protocol
-v3even more screen protocol, program may break
There are further, individual options to the commands.
 

Configuration

CCC.xtp is configured through a UASR configuration file (parameter $2). The configuration file is a Java properties file containing key-value-pairs. The common configuration keys are described with the utility script cfg.itp (see there). Some commands use dedicated configuration keys which are described with the command. Dedicated keys may be placed in the configuration file just like all other keys. Any key (common and dedicated) may be specified or overwritten through the command line using -P<key>=<value> options:

  
  dlabpro CCC.xtp trn myconfig.cfg -P...

-P options always overwrite the configuration file. The heading "uasr." of the keys may be omitted in -P options.

For further information

  • see documentation of util/cfg.itp to learn more about UASR's organization of databases.
  • see documentation of -CFG_get_path to learn how UASR determines paths to the various data sets.
  • see documentation of -CFG_get_flist to learn how UASR obtains file lists of the various data sets.

For each sample being classified the program will output a screen protocol like this:

   - 11/4512 000003: ("OK"4) ... 61 frames5 ... 61 frames5 ... done ("UNK"6).

    Signature :    -FEA_corc(idFeaR,idW,nW,idFeaT)

Description

Parameters

   data    idW   Spectral window definition as returned by {@link -CFG_get_windows}

Return value
