Usage

 dlabpro XVL.xtp <command> <cfgfile> [options]
 dlabpro XVL.xtp help [<command>]
 
Argument/OptionDescription
Arguments
$1: command Command, one of the following:
 
hmm - HMM cross validation (see HMM.xtp)
svm - SVM cross validation (see SVM.xtp)
ccc - CCC cross validation (see CCC.xtp)
cln - reserved Clean up

or "help"
$2: cfgfile UASR configuration file
 
Options
-h, --help Display help and exit
-n Do no actual model training, just say what do
-P<key>=<value> Define or override configuration key-value pairs
-v<n> Verbose level
-v1   minimal screen protocol (default)
-v2extended screen protocol
-v3even more screen protocol, program may break
There are further, individual options to the commands.
 

Operation

Depending on the command (argument $1), XVL.xtp sequentially runs multiple instances of the HMM.xtp, SVM.xtp or CCC.xtp, each time with different, disjoint training and test sets. These sets are drawn from a special cross validation file list according to the following scheme:

Partition 1: testtrain.
Partition 2:train.testtrain.
Partition 3:train.testtrain.
Partition 4:train.testtrain.
Partition 5:train.test

If no cross validation file list is provided, the complete data set (all.flst) will be used instead.

Applications may overwrite -XVL_partition in order to implement custom cross validation schemes.

Configuration

XVL.xtp is configured through a UASR configuration file (argument $2). The configuration file is a Java properties file containing key-value-pairs. The common configuration keys are described with the utility script cfg.itp (see there). There are some dedicated configuration keys which are listed below. Dedicated keys may be placed in the configuration file just like all other keys. Any key (common and dedicated) may be specified or overwritten through the command line using -P<key>=<value> options:

 dlabpro XVL.xtp hmm $UASR_HOME/config/vm/VMV.cfg -P<key>=<value>

-P options always overwrite the configuration file. The heading "uasr." of the keys may be omitted in -P options.

KeyTypeDefaultValue
Dedicated keys to XVL.xtp
uasr.xvl.testnumber  Size of test sets to be draw from the cross validation file list (the remaining files form the training set)
uasr.xvl.flist.statString  Common statistics set for all cross validation parts (individual training sets of parts will be used if omitted)
uasr.xvl.flist.stat_tString  Common linear feature transformation statistics set for all cross validation parts (individual training sets of parts will be used if omitted)
uasr.partnumber  Only process one single partition (omit to process all partitions)
uasr.dlabprostring"dlabpro" dLabPro executable to call
 
Important common keys
uasr.flist.xvalstring Cross validation file list
uasr.flist.sepstring"\t " File list column separator characters
