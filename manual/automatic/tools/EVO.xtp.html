Examples

 dlabpro EVO.xtp --help                        ( display help                                   )
 dlabpro EVO.xtp . -Pcustomize=myevolution.itp ( default configuration and customization script )

In the second example, myevolution.itp is an UASR customization script. The following code snippet shows such a script. The optimization goal of that script is to find a set of two-dimensional, integer, random vectors whose mean vector is (1 2)' and whose covariance matrix consists of multiples of 1/3:

 ## UASR customization script
 ## 
 ## USAGE:
 ##   dlabpro EVO.xtp . -Pcustomize=myevolution.itp
 ##

 ## Individual evaluation function 
 function -evaluate_individual(idX,bShow)  
 {
   statistics iS; 2 2 1 NULL -1 iS -setup;
   data       idM;
   data       idC;
   var        nE;

   idX :idX.dim/2: 2 idX -reshape;
   idX -1 NULL iS -update;
   idM iS -mean;
   idC iS /sn -cov;
  
   :nE  = 2*(idM[0,0] - 1                    )^2;
   :nE += 2*(idM[0,1] - 2                    )^2;
   :nE +=   (idC[0,0] - ent(idC[0,0]*3+0.5)/3)^2;
   :nE +=   (idC[1,1] - ent(idC[1,1]*3+0.5)/3)^2;
   :nE += 2*(idC[1,0] - ent(idC[1,0]*3+0.5)/3)^2;

   bShow if
     idX -print;
     idM -print;
     idC -print;
     data idDetC; :idDetC=det(idC); idDetC -print;
     data idInvC; :idInvC=inv(idC); idInvC -print;
   end

   nE return;
 }

 ## Settings for generating random set of feature vectors 
 "gen.genes"      18   -CFG_set;
 "gen.mvar"        0.5 -CFG_set;
 "gen.mquant"      1   -CFG_set;
 "gen.population" 10   -CFG_set;
 "gen.children"   25   -CFG_set;

 ## Function overwrites
 /disarm -evaluate_individual /disarm -EVO_evaluate_individual =;

 ## EOF

Configuration

EVO.xtp is configured through a UASR configuration file (parameter $2). The configuration file is a Java properties file containing key-value-pairs. The common configuration keys are described with the utility script cfg.itp (see there). There are dedicated configuration keys which are listed in the table below. Dedicated keys may be placed in the configuration file just like all other keys. Any key (common and dedicated) may be specified or overwritten through the command line using -P<key>=<value> options:

  
  dlabpro EVO.xtp . -Pcustomize=myevolution.itp -Pgen.generations=1000

-P options always overwrite the configuration file. The heading "uasr." of the keys may be omitted in -P options.

The following table lists dedicated configuration keys and some important common keys (see section Configuration above).

Function -EVO_population Functions Script Package Keyword Index Function Index Home
    Signature :    -EVO_population(nPopulation,idP)
    Source :    tools/EVO.xtp(75)

Description

Creates the initial population. The initial population is a set of zero vectors. You may overwrite this function to create a custom initial population, e.g.: 
  function -my_population(nPopulation,idP)
  {
    var nGenes; ( "gen.genes" 1 -CFG_get ) nGenes =; # Get number of genes
    -type double nGenes nPopulation idP -array;      # Create zero-initialized population
    1 0 idP -fill;                                   # Initialize population with ones
  }
  /disarm -my_population /disarm -EVO_population =;

The overwritten function should be placed in an UASR customization script (configuration key <uasr.customize>).

Parameters

   number    nPopulation   Number of individuals
   data    idP   Filled with the population

Return value

nothing

Access to Configuration and Global Instances

   <gen.genes>    read

Function -EVO_evaluate_individual Functions Script Package Keyword Index Function Index Home
    Signature :    -EVO_evaluate_individual(idX,bShow)
    Source :    tools/EVO.xtp(157)

Description

Evaluates one individual. The function is expected to compute the "fitness" of an individual. The fitness is a non-negative value. The "fitter" individual has the lower fitness value; the perfect individual has a fitness of 0.

The function is application specific and must be overwritten:

  function -my_evaluate_individual(idX,bShow)
  {
    var nFitness;
    :nFitness = <whatever>;
    bShow if
      idX -print;
    end
    nFitness return;
  }
  /disarm -my_evaluate_individual /disarm -EVO_evaluate_individual =;

The overwritten function should be placed in an UASR customization script (configuration key <uasr.customize>).

Parameters

   data    idX   One vector of genes (= one individual)
   boolean    bShow   TRUE when information on the individual should be printed (used at the end of the evolution to display the result)

Return value

Fitness of the individual; a non-negative value (see above)
